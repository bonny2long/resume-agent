## Writing Style Guidelines

### For Resume Summaries

- First person ("I spent 6 years..." not "Candidate possesses...")
- Start with the trades → tech transition story
- Include 2-3 specific project examples with metrics
- End with technical skills that match the job requirements
- 3 paragraphs maximum
- No buzzwords, no vague claims

### For Cover Letters

- Open with why THIS company interests me (be specific)
- Reference at least one real project with metrics
- Explain how my trades background adds value to THEIR specific challenges
- Keep it conversational but professional
- Don't apologize for non-traditional background - frame it as an advantage

### For Achievements

- Always start with the action: "Built X" not "X was built"
- Include specific metrics: "500+ users" not "many users"
- One achievement per bullet point
- Focus on impact, not just features

## Red Flags to Avoid

If any of these appear in AI-generated content, it's TOO GENERIC:

- Multiple uses of "extensive" or "significant"
- Any sentence starting with "As a seasoned..."
- Phrases like "proven ability to" or "demonstrated expertise in"
- Vague metrics: "numerous," "various," "multiple" (always use actual numbers)
- Passive voice overuse: "was built" instead of "built"
- Generic company praise: "industry leader," "innovative company"

## Quality Checklist

Before using any AI-generated content, verify:

- Includes specific numbers (not vague terms)
- References actual projects by name
- Uses active voice ("I built" not "was built")
- Sounds like me reading it out loud
- No forbidden corporate speak
- No inflated language
- Mentions trades background naturally (when relevant)
- Confident but factual
